# Описание проекта Auto Path Header

## Общая информация
- **Название**: Auto Path Header
- **Автор**: Niklis
- **Тип проекта**: Расширение для Visual Studio Code
- **Версия**: 1.0.0
- **Описание**: Автоматически вставляет относительный путь файла в виде комментария на первой строке

## Функциональность
Расширение предоставляет следующие возможности:
1. Автоматическая вставка пути файла как комментария при открытии файла
2. Поддержка множества языков программирования
3. Предотвращение дублирования комментариев
4. Поддержка локализации (RU/EN)
5. Настраиваемые параметры
6. Обработка ошибок с локализованными сообщениями
7. Автоматическое обновление пути при переименовании/перемещении файла
8. Ручная вставка комментария через палитру команд

## Поддерживаемые языки
### Однострочные комментарии (`//`)
- JavaScript, TypeScript, Java, C, C++, C#, Go, Rust, Swift, Kotlin, PHP

### Комментарии с хэшем (`#`)
- Python, Shell Script, Ruby, Perl, .env

### Блочные комментарии (`/* */`)
- CSS, SCSS, Sass

### SQL комментарии (`--`)
- SQL

### HTML комментарии (`<!-- -->`)
- HTML, XML

## Основные файлы их назначение

### extension.ts
Основной файл активации расширения, который:
- Обрабатывает открытие документов и вставляет комментарии
- Отслеживает переименование файлов и обновляет пути
- Регистрирует команду для ручной вставки комментария

### localization.ts
Содержит локализованные сообщения для русского и английского языков, а также функцию для получения сообщений в зависимости от языка.

### services/config.ts
Предоставляет типобезопасный доступ к настройкам расширения, включая:
- Включение/отключение автоматической вставки
- Язык сообщений
- Обновление при переименовании
- Шаблон форматирования комментариев
- Отключенные языки

### services/inserter.ts
Сервис для вставки и замены комментариев:
- `ensureCommentAtTop` - вставляет комментарий в начало файла
- `replaceTopComment` - заменяет существующий комментарий при переименовании файла

### utils/comments.ts
Утилиты для работы с комментариями:
- Определяет стиль комментариев для различных языков
- Проверяет наличие комментария с путем в строке
- Генерирует комментарий в зависимости от языка и шаблона

## Настройки
- `autoPathHeader.enabled` — включение/отключение автоматической вставки
- `autoPathHeader.language` — язык сообщений (auto/en/ru)
- `autoPathHeader.updateOnRename` — автоматическое обновление комментария при переименовании
- `autoPathHeader.askBeforeUpdate` — запрос подтверждения перед обновлением
- `autoPathHeader.formatTemplate` — шаблон комментария
- `autoPathHeader.disabledLanguages` — список языков, где отключена вставка
