# План интеграционных тестов для расширения Auto Path Header

## Цель
Проверить функциональность расширения в среде VS Code (Extension Development Host) с реальными событиями редактора.

## Основные сценарии

### 1. Активация расширения
- Открытие файла поддерживаемого типа (например, .ts) → расширение активируется
- Выполнение команды `autoPathHeader.insertComment` → расширение активируется

### 2. Автоматическая вставка комментария при открытии файла
#### 2.1 Пустой файл поддерживаемого типа
- Создать/открыть пустой файл (например, `test.ts`)
- Проверить: на первой строке автоматически вставлен комментарий с относительным путем

#### 2.2 Файл с содержимым
- Открыть файл с более чем одной строкой
- Проверить: автоматическая вставка не происходит

#### 2.3 Неподдерживаемый тип файла
- Открыть файл неподдерживаемого типа (например, `.log`)
- Проверить: автоматическая вставка не происходит

#### 2.4 Файл в игнорируемой директории
- Открыть файл в директории из списка `ignoredDirectories` (например, `node_modules/test.js`)
- Проверить: автоматическая вставка не происходит

### 3. Ручная вставка комментария
#### 3.1 Вставка в пустой файл
- Открыть пустой файл поддерживаемого типа
- Выполнить команду `autoPathHeader.insertComment`
- Проверить: комментарий вставлен

#### 3.2 Повторная вставка в тот же файл
- Выполнить команду повторно
- Проверить: дублирование не происходит

#### 3.3 Вставка в неподдерживаемый тип файла
- Открыть файл неподдерживаемого типа
- Выполнить команду `autoPathHeader.insertComment`
- Проверить: выводится сообщение о неподдерживаемом типе

### 4. Обновление комментария при переименовании/перемещении файла
#### 4.1 Обычное переименование (updateOnRename=true, askBeforeUpdate=false)
- Открыть файл с комментарием пути
- Переименовать файл
- Проверить: комментарий обновлен с новым путем

#### 4.2 Переименование с изменением расширения
- Переименовать файл с одного поддерживаемого расширения на другое (например, `.js` → `.ts`)
- Проверить: стиль комментария изменен согласно новому расширению

#### 4.3 Подтверждение обновления (askBeforeUpdate=true)
- Установить `askBeforeUpdate=true`
- Переименовать файл
- Проверить: появляется диалог подтверждения

#### 4.4 Отключение обновления (updateOnRename=false)
- Установить `updateOnRename=false`
- Переименовать файл
- Проверить: комментарий не обновляется

#### 4.5 Переименование в игнорируемую директорию
- Переименовать файл в директорию из `ignoredDirectories`
- Проверить: файл игнорируется согласно настройке

### 5. Настройки расширения
#### 5.1 enabled
- Отключить расширение (`enabled=false`)
- Проверить: автоматическая вставка не работает

#### 5.2 disabledExtensions
- Добавить расширение в `disabledExtensions`
- Проверить: файлы с этим расширением игнорируются

#### 5.3 allowedOnlyExtensions
- Указать список разрешенных расширений
- Проверить: только файлы из списка обрабатываются

#### 5.4 formatTemplate
- Установить пользовательский шаблон
- Проверить: комментарии используют указанный формат

#### 5.5 customTemplatesByExtension
- Установить пользовательский шаблон для конкретного расширения
- Проверить: файлы с этим расширением используют указанный шаблон
- Проверить работу с составными расширениями (например, `.env.local`)
- Проверить работу с конкретными именами файлов (например, `Dockerfile.dev`)

### 6. Поддержка различных типов комментариев
- Проверить корректность формата комментариев для разных типов файлов:
  - Однострочные комментарии: `.js`, `.ts`, `.py`, `.sh` и т.д.
  - Блоковые комментарии: `.css`, `.html`, `.xml` и т.д.
  - Специфические форматы: `.php`, `.json`, `.ini` и т.д.

### 7. Локализация сообщений
- Изменить язык окружения VS Code
- Проверить: сообщения отображаются на соответствующем языке

### 8. Граничные случаи
#### 8.1 Глубоко вложенные файлы
- Открыть файл в глубоко вложенной структуре директорий
- Проверить: корректный относительный путь

#### 8.2 Переименование неоткрытого файла
- Переименовать файл, который не открыт в редакторе
- Открыть файл после переименования
- Проверить: комментарий содержит корректный путь

#### 8.3 Файлы с нестандартными расширениями
- Проверить работу с файлами, имеющими нестандартные или составные расширения

#### 8.4 Файлы с уже существующим комментарием другого формата
- Открыть файл с комментарием, не содержащим пути
- Проверить: корректная обработка

## Технические требования к тестам
- Использовать `@vscode/test-electron` для запуска тестов в среде VS Code
- Создавать временные файлы в тестовой рабочей области
- Проверять содержимое файлов программно
- Учитывать асинхронную природу операций VS Code
- Обеспечивать чистый сброс состояния между тестами
